# Incident Report: INC-5102

## Author: jagadeesan.vg@cognizant.com - 2276259
## Date: 2026-04-22
## Severity: Medium
## Database: salesdb
## Target: SQL Server 2025 VM (10.10.0.5)

## Issue
After successful restore, application team reported "Login failed for user 'app_svc_user'". Application could not connect to the restored database.

## Root Cause
Orphaned users. SQL logins on Managed Instance do not exist on the target VM. Database users had no matching server-level logins.

## Resolution
1. Identified orphaned users:
   SELECT dp.name FROM sys.database_principals dp
   LEFT JOIN sys.server_principals sp ON dp.sid = sp.sid
   WHERE sp.sid IS NULL AND dp.type = 'S'
2. Created missing logins on VM
3. Ran ALTER USER [app_svc_user] WITH LOGIN = [app_svc_user]

## Lesson Learned
Always check and fix orphaned users immediately after every restore.

## Prevention
Added orphaned user fix as mandatory post-restore step in SOP-RESTORE-001.
