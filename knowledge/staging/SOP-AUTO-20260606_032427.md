The restore operation for the database **salesdb** on SQL Server 2025 (Windows VM 10.10.0.5) failed because no `.bak` backup file for `salesdb` was found in the specified Azure Blob Storage location. ## Root Cause Analysis
- **Incorrect or missing blob storage path**: The exact backup `.bak` file URL or blob path for `salesdb` was not found or not correctly specified during the restore command.
- **Backup file naming conventions**: The `.bak` file name for `salesdb` may differ from expected (e.g., spelling, casing, or timestamp/prefix absence) preventing file matching by the automation tool.
- **Blob container or storage account misconfiguration**: The restore tool or operator may have used wrong container name or storage account leading to empty listings.
- **SAS token issues**: The SAS credential used for `RESTORE FROM URL` might be expired, improperly scoped, or lack `list` or `read` permissions, causing inability to discover or access blobs.
- **Network/firewall restrictions**: The Windows VM or the Linux dev VM might have network/firewall rules blocking access to the blob storage endpoint.
- **Automation listing failure**: The automation tool’s blob listing mechanism (used to auto-discover backup files) may have encountered errors or not refreshed cache.
- **Backup not pushed or failed earlier**: The `.bak` file for `salesdb` might not have been successfully backed up and pushed from Azure SQL Managed Instance to blob storage, resulting in a missing file.

## Prevention
- **Implement automatic verification in backup automation** to confirm backup file presence in Blob Storage immediately after backup completion.
- **Centralize and standardize naming conventions** for `.bak` files tied to source DB names and date/timestamp.
- **Monitor SAS token expirations proactively**, and automate renewal before expiry.
- **Add network connectivity monitoring** between VMs and Blob Storage endpoints.
- **Enhance logging and alerting** for blob listing and restore failures on automation.
- Maintain a documented and version-controlled configuration of all Blob Storage paths and SAS tokens used.

## Suggested Tool Improvement
- Enhance `restore_database` tool to:
  - Validate blob path presence before attempting restore, with clearer error messaging.
  - Support auto-refresh or retry for blob listings in case of intermittent failures.
  - Implement automated detection of SAS token expiry and notify or auto-refresh.
  - Support explicit input or override of blob file path for restores to avoid ambiguous discovery.
  - Integrate with Azure RBAC or Managed Identity where possible to reduce SAS token management overhead.
