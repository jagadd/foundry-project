# SOP-AUTO-20260606T041512 : Restore Failure Due to Missing Backup File in Blob Storage

## Classification
- **Category**: RESTORE
- **Severity**: P1
- **Auto-Generated**: Yes
- **Generated On**: 2026-06-06T04:15:12Z

## Problem Description
The database restore operation for the database "Enterprisehrr" failed because the required backup file "Enterprisehrr.bak" could not be found in the Azure Blob Storage container `backupfromazsqlmi` within the storage account `2276259blob`. The restore was attempted via `RESTORE FROM URL` using a SAS credential, executed remotely from a Linux dev VM automation script connecting with pyodbc.

## Root Cause Analysis
Based on the full Azure environment architecture and failure details, the potential root causes are:

1. **Backup File Missing or Misnamed**
   - The backup file `Enterprisehrr.bak` does not exist in the `backupfromazsqlmi` container.
   - Possible naming mismatch: the actual backup filename may differ from `Enterprisehrr.bak` (e.g., case sensitivity, timestamp suffix, database name variant).

2. **Backup File Placement Errors**
   - Backup may have failed or not been pushed properly from the Azure SQL Managed Instance to the blob container.
   - The backup file could be located in a subfolder/path within the blob container not accounted for by the restore automation’s discovery logic.

3. **Blob Storage Access Problems**
   - The SAS token used for `RESTORE FROM URL` may be expired, invalid, or scoped without read/list permissions required to detect or access the backup file.
   - Network restrictions (firewall or NSG) may prevent the SQL Server VM (10.10.0.5) or automation host from reaching the storage account endpoints.

4. **Automation Tool Limitations**
   - The tool's blob listing functionality may fail silently or not enumerate all objects in the container, causing the file to be "undiscovered".
   - Blob container or storage account names might be misconfigured in the automation parameters.

## Resolution Steps
1. **Verify Backup Existence**
   - Log into Azure Portal or use Azure CLI/PowerShell to list blobs in the storage container:
     ```bash
     az storage blob list --account-name 2276259blob --container-name backupfromazsqlmi --output table
     ```
   - Confirm the presence and exact name of the required `.bak` file for `Enterprisehrr`.

2. **Validate Backup Naming Convention**
   - Check naming policies/scripts from SQL MI backups to Blob.
   - Ensure the backup file intended for restore matches exactly `Enterprisehrr.bak` (case-sensitive).
   - If filename includes timestamps or suffixes, update restore automation or rename file accordingly.

3. **Check Backup Push Success**
   - Review backup jobs on Azure SQL Managed Instance.
   - Confirm the last successful backup to blob storage.
   - Re-run or trigger backup push if missing.

4. **Confirm SAS Token Validity and Permissions**
   - Validate the SAS token used in `RESTORE FROM URL` is not expired and includes at least Read (r) and List (l) permissions.
   - Generate a new SAS token if necessary via Azure Portal or CLI:
     ```bash
     az storage container generate-sas --account-name 2276259blob --name backupfromazsqlmi --permissions rl --expiry <future-date> --https-only
     ```
   - Update the automation with the new SAS token.

5. **Verify Network Connectivity**
   - From SQL Server VM (10.10.0.5) and Linux dev VM, test connectivity to Azure Blob storage endpoints.
   - Check Azure NSGs, firewall rules, and routing that could block access.

6. **Validate Automation Listing Logic**
   - Inspect automation logs for blob listing commands and errors.
   - Ensure the tool correctly targets the specified container and does not filter out needed files.

7. **Retry Restore Operation**
   - Once above verifications are complete and any issues addressed, retry restore with confirmed `.bak` file path and updated SAS token.

## Prevention
- Implement monitoring and alerting on backup job success/failures to Azure Blob Storage.
- Implement automated verification post-backup that confirms file presence in blob storage with correct naming.
- Configure SAS tokens with longer validity and automate their refresh if used in automation scripts.
- Regularly audit network connectivity and firewall/NAT rules for the critical paths between SQL MI, blob storage, and SQL Server VM.
- Enhance automation to handle variable backup file names or suffixes and perform robust error handling on blob listing.

## Suggested Tool Improvement
- Enhance `restore_database` tool with configurable blob path and filename pattern matching (e.g., regex) to handle backup files with date/time suffixes.
- Add SAS token expiry detection and automated renewal prompts.
- Integrate pre-restore validation step to verify backup file existence and accessibility, providing detailed diagnostics before starting restore.
- Include verbose logging of blob container listing and permissions check to facilitate faster root cause analysis in future failures.
