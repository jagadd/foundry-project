- The discrepancy between `Finance` and `EnterpriseFinance` implies either a:
  - User input error (requested wrong DB name), or
  - Misconfiguration in automation scripts where the backup and restore names are mismatched.

   - Check the naming standards and ensure the restore database name matches the backup logical DB name unless intentional rename.

4. **Inspect Restore Command Parameters**

## Prevention
- Implement strict naming convention checks in automation pipelines to ensure requested restore DB name matches backup file names and logical DB names.
- Add automated pre-restore validation scripts that:
  - Cross-check the target database name with backup blob file names.
  - Validate SAS tokens and blob access permissions before restore.
- Maintain documentation and runbook for backup naming conventions.
- Monitor SAS token expiry proactively and ensure tokens have sufficient future validity before restore windows.
- Enhance monitoring alerts to capture name mismatch or backup absence early.

## Suggested Tool Improvement
- Enhance the `restore_database` tool to:
  - Automatically list and validate available backup files in the specified blob container before attempting restore.
  - Cross-verify requested database name against backup logical DB names inside .bak files using RESTORE HEADERONLY or RESTORE FILELISTONLY with URL access.
  - Provide clearer error messages indicating name mismatches or missing backup files.
  - Support automatic rename options (`MOVE`) when the restore target name differs from the backup logical DB name, with user confirmation.
  - Integrate SAS token expiry checks and network connectivity tests pre-restore.
