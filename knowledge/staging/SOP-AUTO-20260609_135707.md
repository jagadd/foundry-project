## Problem Description
The automated restore operation for database **EnterpriseSales** from Azure SQL Managed Instance backups stored as `.bak` files on Azure Blob Storage to SQL Server 2025 on the Windows VM (10.10.0.5) failed. The failure occurred due to inability to obtain **exclusive access** to the target database, which had 3 active connections from users `app_user` and `report_svc`. The restore was executed via RESTORE FROM URL using SAS credentials with all pre-flight checks (backup existence, disk space, backup blob verification, target DB state) reported as successful.

## Root Cause Analysis
- **Exclusive Access Conflict**:  
  The target database `EnterpriseSales` was **ONLINE** with **active connections** during the restore attempt:
  - 2 sessions by `app_user` (program `AppPool_v2`)
  - 1 session by `report_svc` (program `ReportService`)
  
  SQL Server requires exclusive access to a database for restore operations, which was blocked by these live connections.

- **No issues with backup file or storage**:  
  - Backup `.bak` exists and was found in Azure Blob Storage (container: `backupfromazsqlmi` in account `2276259blob`) with last modified timestamp recent and valid file size (~1.69 MB).  
  - SAS token for RESTORE FROM URL appears valid, no reported network/firewall/SAS expiry issues.  
  - Disk space (>41 GB free) on target VM was ample relative to backup size.

- **Automation environment**:  
  Restore executed remotely via pyodbc on a Linux dev VM connecting to the Windows VM SQL Server instance. No connectivity or permission errors reported.

## Resolution Steps

1. **Identify and terminate active connections on target database**:  
   On the target SQL Server VM, run the following T-SQL to list and kill interfering sessions:

   ```sql
   -- Identify sessions connected to EnterpriseSales
   SELECT session_id, login_name, host_name, program_name
   FROM sys.dm_exec_sessions
   WHERE database_id = DB_ID('EnterpriseSales');

   -- Terminate sessions (use with caution)
   ALTER DATABASE EnterpriseSales SET SINGLE_USER WITH ROLLBACK IMMEDIATE;

   This sets the database to SINGLE_USER mode, forcibly disconnecting all other connections.

2. **Perform the restore operation after exclusive access is obtained**:

   ```

3. **Set the database back to MULTI_USER mode once restore completes**:

   ```sql
   ALTER DATABASE EnterpriseSales SET MULTI_USER;
   ```

4. **In automation scripting (pyodbc), add pre-restore step to check for active connections**:  
   - Query active sessions on the target database.  
   - If active connections exist, execute termination commands or notify operators.  
   - Only proceed with restore after exclusive access is confirmed.

5. **Coordinate with application and reporting teams**:  
   Schedule restore during maintenance windows where `app_user`, `report_svc`, or other clients are guaranteed disconnected; otherwise, implement connection draining or blocking mechanisms upstream.

## Prevention
- Integrate automated pre-restore checks and forced disconnections or single-user mode toggling in restore tooling to ensure exclusive access before executing restore commands.  
- Communicate and schedule restore jobs during known downtime or maintenance windows to avoid active connection conflicts.  
- Consider implementing SQL Server Resource Governor or workload isolation policies to limit connections before restore operations.  
- Monitor active sessions regularly on critical databases with alerts on unexpected connections during backup/restore windows.

## Suggested Tool Improvement
- Enhance automation tools to **detect active connections** on target database before restore attempts and either:  
  - Automatically force disconnects (with appropriate safeguards), or  
  - Pause the restore operation and alert operators with detailed session info.  
- Add scripting to toggle database to **SINGLE_USER mode with ROLLBACK IMMEDIATE** automatically, followed by restore and then revert to MULTI_USER mode.  
- Implement retries with backoff in automation for restore failures caused by active connections.  
- Log detailed connection metadata and restore operation results for audit and troubleshooting.
